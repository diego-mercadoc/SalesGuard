# SalesGuard - Guia de colaboracion

## Flujo de trabajo

- Trabajar en cambios pequenos y verificables.
- Mantener cada avance listo para integracion.
- Validar el proyecto localmente antes de compartir cambios.

## Buenas practicas de commits

- Antes de hacer un commit:

1. Revisar `git status`.
2. Confirmar que los cambios correspondan al alcance acordado.
3. Verificar que el proyecto siga compilando y arrancando correctamente.
