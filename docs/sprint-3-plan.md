# SalesGuard - Objetivos propuestos para Sprint 3

## Objetivo general

Avanzar de usuarios y autenticacion hacia datasets, para comenzar el flujo principal del proyecto: cargar ventas diarias y guardarlas en base de datos.

## Objetivos tecnicos

1. Crear modelo `Dataset`
2. Crear modelo `DailySales`
3. Agregar migracion de base de datos
4. Crear endpoint para registrar datasets
5. Preparar validacion inicial de datos
6. Definir estructura esperada del CSV

## Entregables esperados

- Tablas `datasets` y `daily_sales`
- Endpoint inicial de datasets
- Validaciones basicas de entrada
- Swagger actualizado con endpoints de datasets
- README actualizado con pasos de uso
