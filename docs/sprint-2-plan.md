# SalesGuard - Objetivos propuestos para Sprint 2

## Objetivo general

Implementar la base de persistencia y autenticacion inicial para pasar de una API base a un backend con usuarios y acceso controlado.

## Objetivos tecnicos

1. Configurar PostgreSQL para desarrollo
2. Elegir ORM entre Prisma o Sequelize
3. Crear modelo `User`
4. Implementar registro de usuario
5. Implementar login con JWT
6. Crear middleware de autenticacion
7. Preparar base de `Dataset` para el siguiente sprint

## Entregables esperados

- Conexion funcional a base de datos
- Primera migracion del proyecto
- Endpoints de registro y login
- Proteccion inicial de rutas privadas
- Documentacion actualizada de auth en Swagger
